# Задача 1. Введение в гетерогенную параллелизацию

## Что такое гетерогенная параллелизация?

Гетерогенная параллелизация - это способ выполнения вычислений, когда мы используем разные типы процессоров одновременно. Обычно это означает что мы используем CPU (центральный процессор) и GPU (графический процессор) вместе, чтобы решать задачи быстрее.

Слово "гетерогенный" означает "разнородный", то есть мы комбинируем устройства с разной архитектурой.

## Различия между параллельными вычислениями на CPU и GPU

### CPU (Central Processing Unit)

- Имеет мало ядер (обычно 4-16 ядер)
- Каждое ядро очень мощное и может выполнять сложные операции
- Хорошо справляется с последовательными задачами
- Имеет большой кэш памяти
- Хорош для задач с ветвлениями (if-else)
- Низкая задержка при выполнении отдельных операций

### GPU (Graphics Processing Unit)

- Имеет много ядер (сотни или тысячи)
- Каждое ядро простое и выполняет базовые операции
- Отлично справляется с одинаковыми операциями над большим количеством данных
- Маленький кэш на каждое ядро
- Плохо работает с ветвлениями
- Высокая пропускная способность памяти

### Таблица сравнения

| Характеристика | CPU | GPU |
|----------------|-----|-----|
| Количество ядер | 4-16 | 1000+ |
| Частота ядра | Высокая (3-5 ГГц) | Низкая (1-2 ГГц) |
| Кэш | Большой | Маленький |
| Латентность | Низкая | Высокая |
| Пропускная способность | Средняя | Очень высокая |
| Тип задач | Сложные, последовательные | Простые, параллельные |

## Преимущества гетерогенной параллелизации

1. **Лучшее использование ресурсов**
   - CPU выполняет сложные части программы
   - GPU обрабатывает большие массивы данных
   - Оба процессора работают одновременно

2. **Увеличение производительности**
   - Некоторые задачи выполняются в 10-100 раз быстрее на GPU
   - Общее время выполнения программы уменьшается

3. **Энергоэффективность**
   - GPU потребляет меньше энергии на операцию при параллельной работе
   - Это важно для серверов и суперкомпьютеров

4. **Гибкость**
   - Можно выбирать устройство под каждую задачу
   - Программа адаптируется к доступным ресурсам

## Примеры реальных приложений

### 1. Машинное обучение и нейронные сети
- TensorFlow, PyTorch используют GPU для обучения моделей
- CPU готовит данные, GPU выполняет матричные операции
- Обучение модели на GPU в 50-100 раз быстрее чем на CPU

### 2. Обработка изображений и видео
- Adobe Premiere, DaVinci Resolve
- GPU применяет фильтры к миллионам пикселей параллельно
- CPU управляет интерфейсом и файлами

### 3. Научные вычисления
- Моделирование молекул
- Симуляция погоды
- Расчеты в физике частиц (CERN)

### 4. Криптография и майнинг
- Подбор хешей на GPU очень эффективен
- Каждый поток проверяет свой вариант

### 5. Компьютерные игры
- GPU рендерит графику
- CPU обрабатывает игровую логику и физику

### 6. Медицина
- Обработка МРТ и КТ снимков
- Реконструкция 3D изображений

## Заключение

Гетерогенная параллелизация позволяет получить максимум производительности, используя сильные стороны каждого типа процессора. CPU хорош для сложных последовательных задач, а GPU - для массивно-параллельных вычислений. Комбинируя их, мы можем эффективно решать современные задачи в области ИИ, графики, науки и многих других областях.
